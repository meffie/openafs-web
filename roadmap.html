Development Roadmap for OpenAFS

This page describes a list of on-going projects and their estimated completion dates.  The purpose of this page is to provide the OpenAFS community a rough idea of direction current development efforts are taking OpenAFS.  The estimated completion dates are not hard commitments.  Instead they are rough estimates that are based upon the OpenAFS Gatekeeper's expectations for the availability of development resources. 

Solaris

  • update client to use system inodes instead of private inode pool (2Q2007)

MacOS

  • 10.5 port (ongoing)
  • improved Finder hinting (1Q2007)

AIX

  • kcred PAG support (1Q2007)

Microsoft Windows

  • Please read the OpenAFS for Windows Status Report published by Secure Endpoints Inc.

Generic

  • test, integrate RxK5 (GSSAPI/Kerberos 5) support including multiple encryption types (3Q2007)
  • Rx IPV6 support (1Q2008)
  • Rx TCP support (2Q2007)
  • PTS rpc changes for alternate principal naming (1/2Q2007)
  • VOS rpc changes (2Q2007)
  • FS rpc changes for new directory object format needed for unicode (1Q2007)

All clients

  • split readonly readwrite cache usage limits (testing needed)
  • disconnected AFS (3Q2007)
  • intermediate fileserver - client as fileserver (3Q2007)

All servers

  • ptserver enhancements for alternate principal names (3Q2007)
  • vlserver enhancements for large volume names (3Q2007)
  • server CellServDB changes for alternate ubik ordering, true multihoming (1Q2008)
  • fileserver directory object handling enhancement to allow for compatible
  • upgrading of directories (2Q2007)

Please send queries and comments regarding this roadmap to openafs-gatekeepers@openafs.org
