OpenAFS for Windows

OpenAFS is the world's foremost location independent file system.  With OpenAFS for Windows, users of Microsoft Windows 2000, XP, 2003 and Vista (32-bit and 64-bit editions) can make the most of this proven data sharing solution.  The OpenAFS Project provides binary installation packages appropriate for both individual users and organizations with thousands of members.

How to Decide Which Version to Install

OpenAFS for Windows releases are labeled Maintenance or Features.  These labels are not related to the quality of the release but to the source code repository branch from which the release is generated.  The Maintenance branch is a version of the source code which is undergoing a slower rate of change.  Where the Features branch may be used to release one or more versions per month, the Maintenance branch is likely to have one release every six or more months.  The Maintenance branch is not used to deliver new functionality nor it is used to support new platforms.  Bugs will always be fixed in the Features branch but may not always be fixed in the Maintenance branch.  Carefully review the feature list of the Features branch before deciding against its use.  Many of the features such as file locking, large file support, and inline bulk status calls are crucial to obtaining good performance with AFS and Microsoft Office applications among others.

The following guidelines should be used for choosing an installer type. If you are upgrading from a previous installation, select the same type of installer, EXE or MSI, as was previously installed on the system. If you are installing for the first time, choose the EXE if you are installing manually and the MSI if you either need to customize the installation package (via a transform) or wish to deploy via Active Directory Group Policy.

Each installer is provided in debug and non-debug versions. The debug versions should be installed only when you are experiencing a problem and you need to assist the OpenAFS developers in debugging the problem. The non-debug version is optimized for performance, does not generate logging information by default, and does not install symbol files.

Digital Signatures

All of the installers are produced and signed by Secure Endpoints Inc.. If the signature is missing do not complete the installation process and send e-mail to openafs-security@openafs.org

Features Release: OpenAFS for Windows 1.5.13a (1.5.1302) **Recommended**

(updated 29 December 2006)

 

Features:

  • All the features of the maintenance release
  • Support for all editions of Windows Vista
  • AFS credentials module for MIT Kerberos for Windows Network Identity Manager
  • Support for 64-bit File Sizes on both 32-bit and 64-bit versions of Microsoft Windows
  • Support for 64-bit Windows XP/2003/R2/Vista
  • Implements Windows Byte Range Locking backed by AFS File Server Locks
  • Implements Inline Bulk Status RPCs which speeds up directory listings
  • Uses GetCapabilities RPCs to probe the server status
  • Logs fs crypt state to the Windows Event Log
  • Supports OutputDebugString debugging of the RX Library
  • New command: fs uuid [-generate]
  • Improved CIFS protocol compatibility
  • Hard Dead and Connection Timeout values restricted to the CIFS Session Timeout value.
  • Improved handling of Windows Standby/Hibernate modes when network access is not available
  • Significant performance improvements when searching directories containing thousands of files or sub-directories
  • Reduced time required for file open operations

  • Windows Vista (X86 and AMD64)
  • Windows 2003 64

  • Windows 2003 including SP1
  • Windows XP including SP2
  • Windows 2000 Workstation and Server

Known Issues:

  • If a *beta* AFS plug-in for Kerberos for Windows is installed, it must be uninstalled before OpenAFS 1.5.13a is installed. Otherwise, an error indicating that the plug-in cannot be installed because the appropriate version of OpenAFS is not installed will be generated.
  • The AFS plug-in for Network Identity Manager provided as part of OpenAFS 1.5.13a requires MIT Kerberos for Windows 3.1 Beta 2 or above.

Maintenance Release: OpenAFS for Windows 1.4.2a (1.4.206)

(released 17 October 2006)

Features:

  • The OpenAFS Client Service provides access seamless access to the AFS file system
    • Access AFS cells via the \\AFS\cellname UNC path
  • Integrates with MIT Kerberos for Windows for Kerberos 5 authentication support
  • The AFS Explorer Shell Extension provides the ability to manipulate AFS objects and permissions from the Windows Explorer Shell
  • The AFS System Tray tool can be used to obtain and monitor AFS authentication credentials (tokens)
  • The Integrated Logon facility allows AFS credentials to be obtained during the Windows logon process enabling Windows Profiles and Redirected Folders to be stored within AFS
  • Automatic configuration of the Windows Internet Connection Firewall
  • Obtains AFS volume database server information via DNS
  • Dynamic Root Volume support
  • Cisco VPN compatible
  • Persistent caches storing up to 1.2GB of data
  • Supports AFS Universal Error Codes
  • Major improvements to AFS SMB Server
  • Improved detection of AFS servers brought back into service
  • Improved maintenance of NAT port mappings
  • Implements Windows Byte Range Locking without use of AFS File Locks
  • Thread priorities are adjusted based upon age of CIFS requst
  • Improved CIFS Server performance due to finer grained locking
  • Improved resource recycling
  • SDK is now installed at \Program Files\OpenAFS\SDK
  • NSIS and WiX installer frameworks updated to the latest versions
  • Improved Kerberos Logon Integration
  • Better handling of dot directory in fs command
  • fs setquota and fs mkmount now behave the same as the UNIX versions
  • Improved Loopback Adapter installer

Documentation:

  • 1.4.2a Release Notes
  • 1.4.2a ChangeLog

Supported Platforms:

  • Windows 2000 Workstation and Server
  • Windows XP
  • Windows XP SP2
  • Windows 2003
  • Windows 2003 SP1
  • Windows 2003 R2

Known Issues:

  • The AFS Servers (bos, file, ...) do not work
  • Directories with large numbers of files or subdirectories will results in very high CPU utilization when searched
  • Application file lock requests are granted without obtaining a lock from the file server.  This can lead to data corruption if two applications are writing to the same file at the same time.

MIT Kerberos for Windows

OpenAFS for Windows depends on MIT Kerberos for Windows (KFW)  to provide Kerberos 5 functionality.  The recommended version of KFW for use with OpenAFS for Windows 1.4.2a is version 2.6.5. The recommended version of KFW for OpenAFS for Windows 1.5.13a is version 3.1.0. KFW 3.1.0 includes the Network Identity Manager which requires an AFS plug-in that is not available in OpenAFS for Windows 1.4.2a.
