OpenAFS for Microsoft Windows®

OpenAFS is the world's foremost location independent file system.  With OpenAFS for Windows users of Microsoft Windows 2000, XP and 2003 (32-bit and 64-bit editions) can make the most of this proven data sharing solution.  The OpenAFS Project provides binary installation packages appropriate for both individual users and organizations with thousands of members.

The following guidelines should be used for choosing an installer. If you are upgrading from a previous installation, select the same type of installer, EXE or MSI, as was previously installed on the system. If you are installing for the first time, choose the EXE if you are installing manually and the MSI if you either need to customize the installation package (via a transform) or wish to deploy via Active Directory Group Policy.

Each installer is provided in debug and non-debug versions. The debug versions should be installed only when you are already experiencing a problem and you need to assist the OpenAFS developers in debuging the problem. The non-debug version is optimized for performance, does not generate logging information by default, and does not install symbol files.
Digital Signatures: All of the installers are produced and signed by Secure Endpoints Inc.. If the signature is missing do not complete the installation process and send e-mail to openafs-security@openafs.org

Current Stable Release: OpenAFS for Windows 1.4.1

(released 23 April 2006)

Features:

  • The OpenAFS Client Service provides access seamless access to the AFS file system
    • Access AFS cells via the \\AFS\cellname UNC path
  • Integrates with MIT Kerberos for Windows for Kerberos 5 authentication support
  • The AFS Explorer Shell Extension provides the ability to manipulate AFS objects and permissions from the Windows Explorer Shell
  • The AFS System Tray tool can be used to obtain and monitor AFS authentication credentials (tokens)
  • The Integrated Logon facility allows AFS credentials to be obtain during the Windows logon process enabling Windows Profiles and Redirected Folders to be stored within AFS
  • Automatic configuration of the Windows Internet Connection Firewall
  • Obtains AFS volume database server information via DNS
  • Dynamic Root Volume support
  • Cisco VPN compatible
  • Persistent caches storing up to 1.2GB of data
  • Supports AFS Universal Error Codes
  • Major improvements to AFS SMB Server
  • Improved detection of AFS servers brought back into service
  • Improved maintenance of NAT port mappings
  • Implements Windows Byte Range Locking without use of AFS File Locks

Documentation:

  • 1.4.1 Release Notes

Known Bugs:

  • The pthread library, afspthread.dll, is not safe to unload. This can cause a crash in applications which load AFS libraries and then unload them when they are no longer required.
  • ANSI filename support is broken.
  • Rapid creation and destruction of SMB virtual circuits can result in a deadlock that will require the AFS service to be terminated via the Task Manager and manually restarted.
  • When the smb_SendPacket routine fails, it may release a reference to the SMB virtual circuit object that it doesn't hold. This can result in a panic condition.
  • There is a known deadlock condition between two locks in the SMB server.
  • fs flushXXX will cause the AFS cache manager to release references to the wrong stat cache entries. This can cause some stat cache entries to never be freed and the resulting file system objects may become inaccessible.
  • volume server failover does not work when mounting the 'root.afs' volume in the default cell. (Freelance mode off.)
  • The help files for the AFS system tray and control panel applications are inaccessible.
  • Mount points to volumes that do not exist can result in a crash of the AFS client service.
  • Over quota and Disk full errors are not returned to the caller.
  • The Integrated Logon DLL can cause applications started by CreateProcessAsUser() to crash. This is known to be a problem with remote installations performed by the System Management Service.
  • Message Box dialogs displayed when a network failure is reported by the bound LAN adapter prevents the automatic restarting of the afsd_service.exe
  • The Integrated Login DLL can cause winlogin.exe to produce a Windows blue screen if it is called with a Windows Domain that does not exist. This can happen during a SysPrep operation.
  • Accessing more than the maximum number of volumes (5000 by default) will result in a panic condition since volume entries are not recycled within the cache manager.
  • The AFS Shell Extension does not always find its resource library resulting in a menu without text strings.
  • An error in the generation of the BackConnectionHostNames registry value can trigger a bug in all versions of Windows that can deadlock the kernel during authentication attempts.
  • The export list from afsauthent.dll is incorrect resulting in third party applications that call pr_End crashing.
  • The winsock library is improperly initialized and released by the RX library causing random network errors in third party applications which load and unload the library as necessary.
  • Directory searches performed by Windows Vista can fail as InfoLevels 0x105 and 0x0106 are not supported but no error is returned.

Stable Release Candidate: OpenAFS for Windows 1.4.2 Release Candidate 3

(released 6 September 2006)

Features:

  • Fixes all known bugs in 1.4.1
  • Thread priorities are adjusted based upon age of CIFS requst
  • Improved CIFS Server performance due to finer grained locking
  • Improved resource recycling
  • Works with Microsoft Vista Build 5536
  • Implements Inline Bulk Status RPCs which speeds up directory listings
  • SDK is now installed at \Program Files\OpenAFS\SDK
  • NSIS and WiX installer frameworks updated to the latest versions
  • Improved Kerberos Logon Integration
  • Better handling of dot directory in fs command
  • fs setquota and fs mkmount now behave the same as the UNIX versions

Development Release: OpenAFS for Windows 1.5.8

(updated 6 September 2006)

Features post-1.4.2:

  • Support for 64-bit Large Files
  • Support for 64-bit Windows XP/2003/R2/Vista
  • Implements Windows Byte Range Locking backed by AFS File Server Locks
  • Uses GetCapabilities RPCs to probe the server status
  • Logs fs crypt state to the Windows Event Log
  • Supports DebugOutputString debugging of the RX Library
  • New command: fs uuid [-generate]
  • Improved CIFS protocol capatibility
  • Hard Dead and Connection Timeout values restricted to the CIFS Session Timeout value.

Documentation:

  • 1.5.8 Release Notes
  • 1.5.8 ChangeLog

Known Bugs:

  • The length of a file can be unexpectedly truncated.
  • Directory searches performed by Windows Vista can fail as InfoLevels 0x105 and 0x0106 are not supported but no error is returned.
  • Accessing more than the maximum number of volumes (5000 by default) will result in a panic condition since volume entries are not recycled within the cache manager.
  • The AFS Shell Extension does not always find its resource library resulting in a menu without text strings.
  • The winsock library is improperly initialized and released by the RX library causing random network errors in third party applications which load and unload the library as necessary.

Compatibility with Microsoft Vista RC1

In order for OpenAFS to be used on Vista RC1, the Windows SMB service must be disabled. This can be done by executing the command sc config smb start= disabled from an elevated Command Prompt session.
