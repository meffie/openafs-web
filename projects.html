Pending and desired OpenAFS projects

Pending projects

The following are items known to be in progress, and, where available, a point of contact.

Disconnected AFS Operations

Provides the ability to use AFS while not connected to a network.

Nested PTS Groups

Allows PTS groups to have other groups as members. Waiting for OpenAFS port.

FreeBSD port

Server ported. No cache manager support yet available.

HP-UX 11 port

Missing RX code reconstructed. Waiting on kernel header from HP.

NetBSD port

No timeline available.

OpenBSD port

Expected by mid-August.

autoconf/automake support

In progress. Contact Derrick Brashear (shadow@dementia.org)

AFSDB DNS resource record support

Substantially complete. Contact Nickolai Zeldovich (kolya@mit.edu)

@sys expansion from list

Substantially complete. Contact Ted McCabe (ted@mit.edu)

Proposed projects

The following are items which have been proposed, but which are not yet in progress. A point of contact is included where appropriate.

Better Documentation

Contact Esther Filderman (moose@cmu.edu)

IPv6 support

Make AFS support IPv6.

Kerberos v5 and multiple encryption types

Add Kerberos v5 and additional encryption type support. Contact Jeff Hutzelman (jhutz@cmu.edu)

Large files and volumes

Support for files over 2gb and large volumes (limit is currently >8gb)

Multiple volume versions

Support for multiple "clones" of a single volume.

Selectable UBIK best host algorithm

Change lowest IP metric to something administrator-selectable.

Volume dump editor

Tool to allow editing volume dumps.

Windows installable file system support

Implement AFS as a Windows file system instead of as a virtual SMB server.
