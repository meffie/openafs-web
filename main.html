What is AFS?

AFS is a distributed filesystem product, pioneered at Carnegie Mellon University and supported and developed as a product by Transarc Corporation (now IBM Pittsburgh Labs). It offers a client-server architecture for file sharing, providing location independence, scalability and transparent migration capabilities for data.
IBM branched the source of the AFS product, and made a copy of the source available for community development and maintenance. They called the release OpenAFS.

18-Oct-2004 - OpenAFS 1.3.73 Released!

OpenAFS 1.3.73 represents the current state of development efforts, and is now the stable Windows 2000/XP/2003 client. For Unix platforms, 1.2.11 is still the stable release. However, 1.3.73 is a milestone on the release path to 1.4.0, and we're looking for serious testing. Note: the earlier 1.3.72 source should be considered withdrawn. The 1.3.72 Windows client matches this one exactly, only a cvs commit error and the resulting distributed source was affected.

10-Aug-2004 - OpenAFS issue for all Windows clients prior to 1.3.70

All versions of IBM/Transarc AFS for Windows and OpenAFS for Windows (prior to 1.3.70) as installed are incompatible with Microsoft Windows XP SP2. Updating to OpenAFS 1.3.70 or modifying the registry configuration is required for safe use on XP SP2.

  • Read the XP SP2 advisory
23-Feb-2003 - Intel, Morgan Stanley, and USENIX fund OpenAFS development.

USENIX has provided a $35,000 grant for OpenAFS, matched evenly by Intel and Morgan Stanley. The OpenAFS Advisory Council accepted this grant on behalf of the OpenAFS community, and funded efforts at MIT, the University of Michigan, and Carnegie Mellon University. The OpenAFS Advisory Council is comprised of representatives from Carnegie Mellon University, MIT, the University of Michigan, Intel Corporation, Morgan Stanley, and IBM Corporation.
