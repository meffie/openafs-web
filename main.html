Welcome to the home of OpenAFS

What is AFS?

AFS is a distributed filesystem product, pioneered at Carnegie Mellon University and supported and developed as a product by Transarc Corporation (now IBM Pittsburgh Labs). It offers a client-server architecture for file sharing, providing location independence, scalability and transparent migration capabilities for data.

What is OpenAFS?

IBM branched the source of the AFS product, and made a copy of the source available for community development and maintenance. They called the release OpenAFS. It's available from the IBM Open Source site.

What is openafs.org, then?

The openafs.org site is for coordination and distribution of ongoing OpenAFS development.

OpenAFS News

15-Oct-2001 - OpenAFS 1.2.2 Released!

OpenAFS 1.2.2 has been released. A bug has been fixed in Linux clients where volume root vnodes were not being garbage-collected which sometimes caused an oops. The unix client now supports a dynamic /afs volume. Several platforms now compute MTU correctly.

22-Sep-2001 - OpenAFS 1.2.1 Released!

OpenAFS 1.2.1 has been released. A bug has been fixed in Linux clients preventing writing of files larger than the cache size. Solaris and Digital UNIX vfsck will now refuse to fsck mounted partitions.

  • Read the list of changes
  • Read the Release Notes
  • Go to the release page

12-Sep-2001 - OpenAFS 1.2.0 Released!

OpenAFS 1.2.0 has been released. Included are fixes for memory leaks in the cache manager and resource tracking errors in the Rx stack, as well as build system enhancements which allow OpenAFS to be installed using make.

  • Read the official release announcement
  • Read the list of changes
  • Go to the release page

23-Aug-2001 - OpenAFS 1.1.1a Released!

OpenAFS 1.1.1a has been released. This version includes several updates to the Windows NT and 98 installers, as well as enhancements to the Windows 98 client to allow unauthenticated access. There are no changes to any non-Windows version; 1.1.1 is still current for all non-Windows platforms.

  • Go to the release page

1-Jul-2001 - OpenAFS bug submission interface available.

A bug database is being implemented. The web frontend is not yet available, but bugs, patches and enhancements can be submitted to openafs-bugs@openafs.org .

18-Jun-2001 - openafs.org cell

The openafs.org AFS cell is now available. It currently contains copies of OpenAFS source and binary releases, and is expected to grow to contain additional content in the future.
