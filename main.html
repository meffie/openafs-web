What is AFS?

AFS is a distributed filesystem product, pioneered at Carnegie Mellon University and supported and developed as a product by Transarc Corporation (now IBM Pittsburgh Labs). It offers a client-server architecture for file sharing, providing location independence, scalability and transparent migration capabilities for data.
IBM branched the source of the AFS product, and made a copy of the source available for community development and maintenance. They called the release OpenAFS.

22-Mar-2005 - AFS & Kerberos Best Practices Workshop

The second annual AFS & Kerberos Best Practices Workshop will be held June 20-24, 2005 at Carnegie Mellon University. The workshop will include two days of tutorials and three days of talks including keynote speaker Mike Kazar, one of the original developers of AFS. For more information or to submit a talk or paper, see http://www.pmw.org/afsbpw05/.

28-Apr-2005 - OpenAFS 1.3.82 Released!

OpenAFS 1.3.82 represents the new stable Windows release, and includes a multitude of bug fixes as well as several new features. For Unix platforms, 1.3.82 includes several important fixes for bugs in previous versions of OpenAFS 1.3.

6-Nov-2004 - OpenAFS 1.2.13 Released!

OpenAFS 1.2.13 has a critical bugfix for an issue where fileservers can hang or crash when a client is creating too many connections. No new features are present.

  • Read the 1.2.13 release announcement
  • Go to the release page

10-Aug-2004 - OpenAFS issue for all Windows clients prior to 1.3.70

All versions of IBM/Transarc AFS for Windows and OpenAFS for Windows (prior to 1.3.70) as installed are incompatible with Microsoft Windows XP SP2. Updating to OpenAFS 1.3.70 or modifying the registry configuration is required for safe use on XP SP2.

  • Read the XP SP2 advisory
