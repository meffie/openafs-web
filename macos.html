OpenAFS for MacOS X

OpenAFS is the world's foremost location independent file system.  With OpenAFS for MacOS users of MacOS 10 can make the most of this proven data sharing solution.  The OpenAFS Project provides binary installation packages appropriate for both individual users and organizations with thousands of members.

Maintenance Release: OpenAFS 1.4.11 for MacOS 10.5 (Leopard) **Recommended**

(released 14 Jul 2009)

Features since 1.4.10:

  • Network stack leaks fixed.
  • Shutdown issues due to double-freeing memory have been fixed.
  • The 1.4.8 security vulnerability does not apply to MacOS 10.5

Downloads:

  • MacOS 10.5 (Leopard) Universal package

Known Issues:

  • AFS icon does not appear on user desktop. (Bug 75382)
  • Finder cannot move items by dragging across AFS volumes. (Bug 34255)
  • Finder does not display the AFS volume on the Desktop unless the user has selected 'Connected servers" in Finder->Preferences->General.
  • Finder consumes excess resources while AFS running (PowerPC only, no details yet). (Bug 87482)

Maintenance Release: OpenAFS 1.4.11 for MacOS (Panther) **Recommended**

(released 20 Jul 2009)

Features:

  • The OpenAFS Client Service provides access seamless access to the AFS file system
  • Integrates with MIT Kerberos for Mac for Kerberos 5 authentication support
  • Supports AFS Universal Error Codes
  • Cisco VPN compatible
  • Improved detection of AFS servers brought back into service
  • Persistent caches storing up to 20GB of data

Known Issues:

  • When authentication state changes, Finder may cache stale data (files appear unaccessible or do not appear)

Documentation:

Maintenance Release: OpenAFS 1.4.12pre3 for MacOS 10.5 (Leopard) Prerelease

(released 11 Feb 2010)

Features since 1.4.11:

  • /afs enumeration in finder is greatly improved.
  • File metadata (stat() results) are no longer leaked to unauthorized users.
  • Avoid blindly trusting that data fetches will return only as much data is requested.
  • Improved fileserver host tracking.
  • A PreferencesPane for configuring OpenAFS is now included.
  • launchdaemon is now used to start OpenAFS.
  • Fileserver startup issues (VL registration and retried starts) fixed.
  • Allow caches on large partitions.
  • Return network unreachable errors in Rx immediately in userspace also.
  • Installer.app now prompts for cell name.
  • Fix Rx resource management issues/races.
  • Avoid spurious DNS lookups in dynroot.
  • Correct shutdown-time memory cleanup.
  • Fix fstrace for 64 bit kernels.
  • Ensure vos operations do not clobber existing volume IDs.

Documentation:

  • 1.4.12pre3 Release Notes

Downloads:

  • MacOS 10.5 (Leopard) Universal package

Known Issues:

  • AFS icon does not appear on user desktop. (Bug 75382)
  • Finder cannot move items by dragging across AFS volumes. (Bug 34255)
  • Finder consumes excess resources while AFS running (PowerPC only, no details yet). (Bug 87482)

Third Party AFS Tools for MacOS X

  • AFS Commander: an OSX preferences pane to simplify management of the OpenAFS client.
  • A collection of third party tools for use with AFS on MacOS X is maintained by bil hayes at http://www.ibiblio.org/macsupport/afs/afstools.html
  • AFS Tokens is a gui application for Mac and Linux. This application allows you to view, obtain, and destroy tokens. You can also view your afs group membership and view/create/edit/destroy owned groups. https://forge.cornell.edu/sf/projects/afs_tokens
